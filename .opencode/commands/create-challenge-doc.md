description: CTF 题目落盘 — 从网站读取题目、下载原始附件（不解压）、生成忠实转录的本地分析文档。绝不分析/解读/发散
---

## 角色与目标

把网站上的 CTF 题目**忠实落盘**到本地（分析文档 + 原始附件），为后续解题做准备。

**唯一边界：只落盘，不解题。** 分析、解读、构造利用都是解题阶段的事，本命令不做。

## 输入（$ARGUMENTS）

- 网站题目页 URL
- （如需登录）登录方式 + 凭证
- 用户指定的解题要求（白盒/黑盒、是否禁 writeup、是否要自动提交验证等）

$ARGUMENTS

## 输出

- 文档：`docs/分析/<分类>/<短名>.md`
- 附件：`docs/分析/<分类>/Attachments/<短名>/`（**原始文件，不解压**）
- 凭证（若有）：`.privacy-data/<比赛>/`（gitignored）

## 核心原则（8 条防越界清单 — 强制）

| # | 原则 |
|---|------|
| 1 | **只转录页面原文，不分析**：不写源码分析/架构/bot 行为/API 枚举——那是解题阶段的事 |
| 2 | **不解读、不加注解**：原文照录，不擅自解释术语/正则/编码（如不把 `[\x20-\x7e]` 注成"可打印 ASCII"） |
| 3 | **不写未验证断言**：没观察到的数值（如初始分值）、没验证的机制（如 token 轮换）一律不写，不把推断当事实 |
| 4 | **附件只下载原始文件，不解压**：解压/阅读内容是解题阶段 |
| 5 | **不用未解释黑话**：避免 tarball/slug/URL 编码等术语，用文件全名或通俗说法 |
| 6 | **凭证不进 docs/ 且从简**：token/密码绝不写进 `docs/`（进 git）；存 `.privacy-data/`（gitignored）；文档只写位置和登录方式。凭证文件从简——一个登录凭证即可，不做冗余快照/多份副本/长说明 |
| 7 | **跨平台命名**：文件/目录名避开 Windows 禁用字符 `< > : " / \ \| ? *`；题目名含特殊字符时用安全短名（如挑战子域名），文档内记真实标题 |
| 8 | **解题要求只记用户指定的**：不自行添加 |

## 执行流程

1. （需登录时）用凭证登录；凭证存 `.privacy-data/<比赛>/`（不写进 docs/）
2. 读取题目页面
3. 转录元信息（标题/分类/作者/分值/解题数/Flag 格式——如实，不臆断数值）
4. 转录描述原文（Description/Note 等）
5. 转录连接信息（靶机/提交平台 URL）
6. 下载原始附件（Attachments 区、Download config 等所有页面可下载材料），**不解压**
7. 记录提交入口（页面可见的 admin bot/flag 框等，无则省略）
8. 写解题要求节（仅用户指定）
9. 写登录与凭证节（方式+位置，不含值；无需登录则省略）
10. 执行落盘后自检清单

## 落盘后自检清单（对照 8 条原则，全部 ✅ 才算完成）

- [ ] 文档无分析/解读内容？
- [ ] 无自行加注解？
- [ ] 无未验证断言/臆断数值？
- [ ] 附件未被解压？
- [ ] 无未解释黑话？
- [ ] 凭证未写进 docs/（只在 .privacy-data/）且从简（无冗余快照/长说明）？
- [ ] 文件名无 Windows 禁用字符？
- [ ] 解题要求无自行添加？

## 输出文档结构模板（各节按需，题目无对应内容则省略）

```
# 分析-<短名>（题目 <真实标题>）
> 来源 / 平台

## 题目元信息（table：标题/分类/作者/分值/解题数/Flag 格式）
## 题目描述（官网原文）
## 连接信息
## 附件（仅下载，未解压未分析）
## 提交入口（页面可见，无则省略）
## 解题要求（用户指定）
## 登录与凭证（方式+位置，不含值；无需登录则省略）
```
