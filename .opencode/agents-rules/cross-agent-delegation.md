## 跨 Agent 自动委派

分析中遇到**不属于你专长领域**的子问题，用 **Task 工具**自动委派给对应专家 agent——不要自己硬啃、不要停下来问用户。

| 遇到的子问题 | 委派给 | 你要提供的上下文 |
|-------------|--------|-----------------|
| 密码学破解/求解（RSA/格/AES/ECC/古典/对称/哈希） | `crypto-analysis` | 算法类型、已知参数（n/e/c/曲线/密文等）、目标（求 flag/明文/密钥） |

**委派方法**：`Task(subagent_type: "<上表专家>", description: "<详细描述：已知什么、要求返回什么>")`

**规则**：
- 把你已发现的**全部相关信息**传给子 agent（算法位置、参数、密文等），不要只说"帮我解个 RSA"
- 要求子 agent 返回**具体的 flag 或明文**，不是"建议"
- 拿到返回结果后**整合进你的分析继续**，不要停下来等用户
- 如果子 agent 也解决不了，把它的反馈纳入你的分析，继续其它方向
